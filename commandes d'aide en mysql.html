Liste des commandes SQL

Introduction

Tout au long des différentes commandes présentées et leurs exemples, la table "shows" sera utilisée dont voici la structure et les données:

Commandes

La plupart des commandes et mot-clés présentés ci-dessous en tant que titre (en bleu) sont également des liens vers le manuel de référence de MySQL 8.0.

USE

L'instruction USE indique à MySQL d'utiliser la base de données nommée comme base de données (courante) par défaut pour les instructions suivantes.

La base de données nommée reste la valeur par défaut jusqu'à la fin de la session ou jusqu'à ce qu'une autre instruction USE soit émise

Exemple

La requête ci-dessous indique à MySQL d'utiliser la base de données shows_db

USE 'shows_db';

SELECT

La commande SELECT permet de retourner sous la forme d'un tableau un ou plusieurs colonnes d'une ou plusieurs table.

SELECT colonne_1, colonne_2 FROM nom_du_tableau;

Exemple La requête ci-dessous retourne le nom et le nombre de saisons de tous les enregistrements de la table "shows".

SELECT name, nb_seasons FROM shows;

voir le résultat.

Afficher toutes les colonnes

Si l'on désire afficher toutes les colonnes d'une table, il est possible d'utiliser le caractère "*" au lieu de lister tous les noms des colonnes.

Exemple La requête ci-dessus retourne toutes les colonnes ainsi que tous les enregistrements de la table "shows".

SELECT * FROM shows;

Remarque Le caractère "*" ne doit pas être utilisé dans des requêtes émises par des applications où la performance est un critère important

DISTINCT

La commande DISTINCT permet d'éviter les redondances dans les résultats affichés lors de l'utilisation de la commande SELECT. 

SELECT DISTINCT colonne_1, colonne_2 FROM nom_du_tableau;

Exemple La requête ci-dessous retourne la liste de toutes les chaînes 

SELECT DISTINCT channel FROM shows;

ORDER BY

La commande ORDER BY permet d'obtenir le résultat d'une requête trié selon un ou plusieurs critères

SELECT colonne_1, colonne_2,...FROM nom_du_tableau
ORDER BY colonne_1, colonne_2,....;

Remarque: Il est possible de trier par ordre décroissant en ajoutant DESC après le nom de la colonne

Exemple 1 La requête ci-dessous retourne le nom, le nombre de saisons et le nombre d'épisodes triées par le nombre de saisons.

SELECT name, nb_seasons, nb_episodes FROM shows
ORDER BY nb_seasons;

voir le résultat.

Exemple 2 La requête ci-dessous retourne le nom, le nombre de saisons et le nombre d'épisodes triées par le nombre de saisons puis par le nombre d'épisodes par ordre décroissant.

SELECT name, nb_seasons, nb_episodes FROM shows
ORDER BY nb_seasons, nb_episodes DESC;

voir le résultat.

WHERE

La commande WHERE permet de retourner les différents enregistrements respectant une ou plusieurs plusieurs conditions.

SELECT colonne_1, colonne_2,... FROM nom_du_tableau
WHERE condition;

La ou les condition(s) s'écrivent avec des opérateurs logiques (AND, OR), des opérateurs de comparaison (=,>, <=,...).

Lien vers la liste des opérateurs

Exemple La requête ci-dessous retourne le nom et l'année de sortie des séries sorties en 2013.

SELECT name, year_broadcast FROM shows
WHERE year_broadcast = 2013;

BETWEEN

L'opérateur BETWEEN est utilisé dans les requêtes utilisant la commande WHERE. Celui-ci permet de retourner des enregistrements compris entre 2 valeurs d'un colonne.

SELECT colonne_1, colonne_2,... FROM nom_du_tableau
WHERE colonne_1 BETWEEN val_1 AND val_2;

Exemple La requête ci-dessous retourne le nom, le nombre de saisons et le nombre d'épisodes des séries dont le nombre d'épisodes est compris entre 60 et 70.

SELECT name, nb_seasons, nb_episodes FROM shows
WHERE nb_episodes BETWEEN 60 AND 70;

IN

L'opérateur IN est utilisé dans les requêtes utilisant la commande WHERE pour vérifier si une colonne est égale à une des valeurs comprise dans un set de valeurs déterminées.

SELECT colonne_1, colonne_2,... FROM nom_du_tableau
WHERE colonne_1 IN (val_1, val_2,...)

Exemple La requête ci-dessous retourne le nom, le nombre de saisons et la chaîne des séries diffusées originalement sur les channels "HBO" ou "AMC":

SELECT name, nb_seasons, channel FROM shows
WHERE channel IN ('HBO', 'AMC');

LIKE

L'opérateur LIKE est utilisée dans les requêtes utilisant la commande WHERE. Il permet de retrouver un modèle (p. ex. un caractère ou une suite de caractères).

SELECT colonne_1, colonne_2,... FROM nom_du_tableau
WHERE colonne_1 LIKE modèle;

L'opérateur LIKE est très couramment utilisé avec les caractères Joker suivants:

% remplaçant tous les autres caractères. _ remplaçant un caractère

Exemple 1 La requête ci-dessous retourne le nom et la channel des séries contenant la suite de caractères "gen" n'importe où dans le nom de la série.

SELECT name, channel FROM shows
WHERE name LIKE ('%gen%');

Exemple 2 La requête ci-dessous retourne le nom et la channel des séries dont le nom commence par "The".

SELECT name, channel FROM shows
WHERE name LIKE ('The%');

Opérateurs logiques AND, OR, NOT

Les opérateurs logiques AND et OR peuvent être utilisées dans les requêtes utilisant la commande WHERE pour combiner plusieurs conditions. L'opérateur logique NOT permet de retourner des enregistrements lorsque la condition qui suit le NOT n'est pas remplie

syntaxe générale avec AND

SELECT colonne_1, colonne_2,... FROM nom_du_tableau
WHERE condition_1  AND condition_2;

Exemple 1 La requête ci-dessous retourne le nom, l'annee de première diffusion et le score des utilisateurs des séries diffusées en 2013 et dont le score est supérieur à 8.5.

SELECT name, year_broadcast, user_score FROM shows
WHERE year_broadcast = 2016 AND user_score > 8.5;

Exemple 2 La requête ci-dessous retourne le nom, l'annee de première diffusion et le score des utilisateurs des séries diffusées qui n'ont pas été diffusées en 2008 et dont le score est supérieur à 9.

SELECT name, year_broadcast, user_score FROM shows
WHERE NOT year_broadcast = 2008 AND user_score > 9;

GROUP BY

La commande GROUP BY est utilisée pour grouper plusieurs résultats et utiliser une fonction de totaux sur un groupe de résultat. Cette commande est donc habituellement accompagnée de fonctions d'agrégats tels que COUNT(), SUM(), MAX(),.... .

SELECT colonne_1, fonction(colonne_1),... FROM nom_du_tableau
GROUP BY colonne_1;

Fonctions d'agrégations

Voici une liste non exhaustive de fonctions d'agrégats utilisées avec la commande GROUP BY

  • COUNT() : Permet de compter le nombre d'enregistrements dans une table
  • SUM() : Permet de calculer la somme totale sur un ensemble d'enregistrement
  • AVG(): Permet de calculer la moyenne sur un ensemble d'enregistrement
  • MIN(): Permet de retourner la valeur minimum sur un ensemble d'enregistrement
  • MAX(): Permet de retourner la valeur maximum sur un ensemble d'enregistrement

Exemple 1 La requête ci-dessous retourne les chaînes et le nombre de série diffusées sur chaque chaîne.

SELECT channel, COUNT(id) FROM shows
GROUP BY channel;

Exemple 2 La requête ci-dessous retourne les chaînes et la moyenne des scores des séries de chaque chaîne.

SELECT channel, AVG(user_score) FROM shows
GROUP BY channel;

AS

Il est possible d’utiliser des alias pour renommer temporairement une colonne ou une table dans une requête. Ces Alias sont pratiques pour avoir un nom plus facilement identifiable. Le mot-clé AS est utilisé pour créer ces Alias.

A noter qu’un Alias peut être créé pour une colonne ou une table

SELECT colonne_1 AS C1, colonne_2 FROM nom_du_tableau ;

Exemple La requête ci-dessous retourne les channels et le nombre de série diffusées sur chaque chaîne, cette dernière colonne étant renommée nb_shows :

SELECT channel, COUNT(id) AS nb_shows FROM shows
GROUP BY channel;

HAVING

La condition HAVING en SQL est presque similaire à WHERE à la seule différence que HAVING permet de filtrer en utilisant des fonctions telles que SUM(), COUNT(), AVG(), MIN() ou MAX().

SELECT colonne_1, fonction(colonne_1),... FROM nom_du_tableau
GROUP BY colonne_1
HAVING condition;

Exemple La requête ci-dessous retourne les chaînes et le nombre de séries diffusées sur chaque chaîne, cette dernière colonne étant renommée nb_shows:

SELECT channel, COUNT(id) AS nb_shows FROM shows
GROUP BY channel
HAVING COUNT(id) >=3;

INSERT INTO

La commande INSERT INTO permet d'insérer des données dans une table. Il est possible d'insérer une seule ou plusieurs lignes à la fois.

Il y a 2 syntaxes principales :

  • Insérer une ligne en spécifiant toutes les colonnes Cette syntaxe offre la possibilité d'entrer toutes les valeurs d'une ligne. Elle et a l'avantage de ne pas devoir entrer le nom des colonnes. Il faut toutefois respecter l'ordre des colonnes.
INSERT INTO table VALUES ('valeur 1', 'valeur 2', ...);
  • Insérer une ligne en spécifiant seulement des colonnes souhaitées. Cette syntaxe offre la possibilité d'entrer seulement des valeurs dans certaines colonnes. Elle a l'avantage de ne pas remplir des valeurs dans chaque colonne.
INSERT INTO table (nom_colonne_1, nom_colonne_2, ...) VALUES ('valeur 1', 'valeur 2', ...);

Il est également possible de remplir plusieurs lignes à la fois:

INSERT INTO table (nom_colonne_1, nom_colonne_2, ...) VALUES
('valeur 1_1', 'valeur 1_2', ...),
('valeur 2_1', 'valeur 2_2', ...),
('valeur 3_1', 'valeur 3_2', ...);

UPDATE

La commande UPDATE permet d’effectuer des modifications sur une ou plusieurs lignes existantes.

Cette commande est très souvent utilisée avec WHERE pour spécifier sur quelles lignes doivent porter la ou les modifications.

UPDATE table
SET nom_colonne = 'nouvelle valeur'
WHERE condition;

Exemple La requête ci-dessous modifie le nom de la chaîne 'Canal+' en 'C+'

UPDATE shows
SET channel = 'C+'
WHERE channel ='Canal+';

DELETE

La commande DELETE en SQL permet de supprimer une ou plusieurs lignes dans une table.

En utilisant cette commande associé à WHERE il est possible de sélectionner les lignes concernées qui seront supprimées.

DELETE FROM table
WHERE condition;

Exemple La requête ci-dessous supprime de la table toutes les séries de la chaîne 'HBO'

DELETE FROM shows
WHERE channel = 'HBO' ;

ALTER TABLE

La commande ALTER TABLE permet de modifier la structure d'une table existante. Il est ainsi possible d’ajouter une colonne, d’en supprimer une ou de modifier une colonne existante, par exemple pour changer le type.

la syntaxe de base de cette commande est la suivante

ALTER TABLE nom_table
instruction

où instruction désigne une des commandes détaillées ci-dessous:

Ajouter une colonne

La syntaxe pour ajouter une colonne à une table est la suivante

ALTER TABLE nom_table
ADD nom_colonne type_donnees

Exemple La requête ci-dessus ajoute dans la table "shows" une colonne nommée meta_score contenant des entiers .

ALTER TABLE shows
ADD meta_score int;

Remarque: Sans autre paramètre ajouté à cette commande, la colonne est créée tout à droite de la table

Supprimer une colonne

La syntaxe pour supprimer une colonne d'une table est la suivante

ALTER TABLE nom_table
DROP COLUMN nom_colonne

Exemple La requête ci-dessus supprime la colonne "user_score" de la table "shows".

ALTER TABLE shows
DROP COLUMN user_score;

Modifier une colonne

La syntaxe pour modifier le type d'une colonne dans table est la suivante

ALTER TABLE nom_table
MODIFY nom_colonne type_donnees;

Exemple La requête ci-dessus modifie le type de la colonne "user_score" de la table "shows".

ALTER TABLE shows
MODIFY year_broadcast int;

Renommer une colonne

La syntaxe pour renommer une colonne d'une table est la suivante

ALTER TABLE nom_table
CHANGE colonne_ancien_nom colonne_nouveau_nom type_donnees;

Exemple La requête ci-dessus modifie le type de la colonne "user_score" de la table "shows".

ALTER TABLE shows
CHANGE year_broadcast year_begin year;
